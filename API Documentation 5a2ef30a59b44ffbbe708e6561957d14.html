Introduction

Authentication

Requests are authenticated by your API key, which must be appended to the end of the URL as a parameter named “code”. For example:

$ curl "https://[api url]/[example]?code=[YOUR_API_KEY]"

Note that your API key is what identifies you and your usage, so please don’t share it with anyone outside of your organization.

Errors

IncludeHealth uses common HTTP response codes to indicate the status of the API request.

In case of an error, IncludeHealth API will return the correspondent code.

Session Webhooks

To notify integration partners when sessions are completed, we will need to implement an outbound webhook call to the integration partner’s own API. We’ll need a URL to call out to, and I propose sending a simple payload containing just the session Id (ours).

{
	"SessionId": "11112222-3333-4444-5555-666677778888"
}

The partner can then take this Id and call back into our API to retrieve the report. This gets us the same security we already have on our public API, requiring an API/Azure Function key to call the endpoint, and using that key to identify the calling org. This prevents us from having to implement multiple different outbound security models to support different partners.
